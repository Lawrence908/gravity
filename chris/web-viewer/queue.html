Set up a run: one heavy “star” at the centre and many lighter particles. When you hit Run, the sim will queue and you can watch it in the viewer when it’s done.

Flat disk (2D) or full 3D space. 3D looks more like a galaxy. Disk = particles start in a ring with orbits; cloud = more random spread. Disk is best for spiral-like runs.
More particles = richer picture, slower run. 1000–5000 is a good range.
How long the sim runs. More steps = longer evolution (e.g. 20k–50k for orbits).
Smaller = more accurate but slower. 0.01 is usually fine; drop to 0.005 if things look unstable.
Ratio 1000 : 1 — m_particle = 0.001 (star = 1)
Star: 1 M☉ ≈ 1.99×10³⁰ kg · Particle: ≈ 1.99×10²⁷ kg
How much heavier the star is than each particle. Higher ratio = lighter particles = orbits stay stable longer. Sizes in the viewer scale with mass. Inner edge of the ring (disk only). Keep above 0.5 so particles don’t start on plunging orbits. Outer edge: particles start somewhere inside this distance from the star. Larger = start further out. Smooths gravity when particles get very close so nothing “blows up.” 0.05–0.1 is typical. M_halo = 0 (off) Extra invisible mass that holds the disk in orbit (like in real galaxies). 0 = off; try 5–20 for spiral-style runs. How spread out the dark matter is. Often set to about half of r_max. Same number = same initial layout every time (handy for comparing runs). Leave empty to use suggested name
GPU = faster when you have many particles (needs a CUDA GPU). Collisions = particles can merge when very close; off = they pass through each other. How close two bodies must get to merge. Auto = a small safe value (only very close pairs merge).
